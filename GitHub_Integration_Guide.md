# دليل ربط الموقع مع GitHub

## مقدمة
هذا الدليل يساعدك على ربط موقع ROMAVEN LTD مع GitHub وإعداده للنشر على Hostinger.

## الخطوة 1: إنشاء حساب على GitHub
1. اذهب إلى [github.com](https://github.com)
2. انقر على "Sign up" لإنشاء حساب جديد
3. أكمل عملية التسجيل بالبيانات المطلوبة
4. تحقق من بريدك الإلكتروني

## الخطوة 2: إنشاء مستودع جديد (Repository)
1. بعد تسجيل الدخول، انقر على "+" في الزاوية العلوية اليمنى
2. اختر "New repository"
3. أدخل اسم المستودع (مثلاً: romaven-website)
4. اترك المستودع عاماً (Public) أو خاصاً (Private) حسب تفضيلك
5. انقر على "Create repository"

## الخطوة 3: تثبيت Git على جهازك
1. قم بتحميل Git من [git-scm.com](https://git-scm.com/downloads)
2. ثبت Git على جهازك باتباع التعليمات
3. افتح موجه الأوامر (Command Prompt) أو PowerShell

## الخطوة 4: إعداد Git
1. افتح موجه الأوامر
2. قم بتعيين اسمك:
   ```
   git config --global user.name "اسمك"
   ```
3. قم بتعيين بريدك الإلكتروني:
   ```
   git config --global user.email "بريدك_الإلكتروني"
   ```

## الخطوة 5: إعداد مفتاح SSH (اختياري ولكن موصى به)
1. افتح موجه الأوامر
2. أدخل الأمر:
   ```
   ssh-keygen -t rsa -b 4096 -C "بريدك_الإلكتروني"
   ```
3. اضغط Enter لقبول الموقع الافتراضي
4. أدخل عبارة مرور (passphrase) أو اضغط Enter لتخطيها
5. اعرض المفتاح العام:
   ```
   cat ~/.ssh/id_rsa.pub
   ```
6. انسخ المفتاح العام
7. اذهب إلى GitHub → Settings → SSH and GPG keys
8. انقر على "New SSH key"
9. الصق المفتاح العام وأضف عنواناً له
10. انقر على "Add SSH key"

## الخطوة 6: إضافة ملفات المشروع إلى Git
1. افتح موجه الأوامر في مجلد المشروع
2. قم بتهيئة مستودع Git:
   ```
   cd "c:\Users\Admin\Desktop\ltd 2"
   git init
   ```
3. أضف جميع الملفات:
   ```
   git add .
   ```
4. قم بإنشاء أول التزام (commit):
   ```
   git commit -m "Initial commit"
   ```

## الخطوة 7: ربط المستودع المحلي بـ GitHub
1. انسخ عنوان المستودع من GitHub (باستخدام HTTPS أو SSH)
2. أضف المستودع البعيد:
   ```
   git remote add origin عنوان_المستودع
   ```
3. ادفع الملفات إلى GitHub:
   ```
   git push -u origin master
   ```
   أو إذا كنت تستخدم الفرع الرئيسي باسم main:
   ```
   git push -u origin main
   ```

## الخطوة 8: ربط GitHub مع Hostinger
1. قم بتسجيل الدخول إلى حساب Hostinger
2. اذهب إلى "Hosting" → "Manage"
3. ابحث عن "Git" في لوحة التحكم
4. انقر على "Create new Git instance"
5. أدخل عنوان مستودع GitHub الخاص بك
6. اختر الفرع (branch) الذي تريد استخدامه (عادة master أو main)
7. حدد المجلد الذي تريد نشر الموقع فيه (public_html)
8. انقر على "Create"

## الخطوة 9: النشر التلقائي
1. بعد إنشاء مثيل Git، سيقوم Hostinger بسحب الملفات من GitHub
2. يمكنك إعداد النشر التلقائي عند كل دفع جديد إلى GitHub
3. للقيام بذلك، قم بإنشاء Webhook في GitHub:
   - اذهب إلى مستودع GitHub الخاص بك
   - انقر على "Settings" → "Webhooks"
   - انقر على "Add webhook"
   - أدخل عنوان URL الذي يوفره Hostinger
   - اختر "Just the push event"
   - انقر على "Add webhook"

## استكشاف الأخطاء وإصلاحها

### المشكلة: فشل ربط GitHub مع Hostinger
الحل:
- تأكد من أن عنوان المستودع صحيح
- تحقق من أن الفرع المحدد موجود في GitHub
- تأكد من أن لديك الصلاحيات اللازمة للوصول إلى المستودع

### المشكلة: لا يتم تحديث الموقع بعد دفع التغييرات إلى GitHub
الحل:
- تحقق من أن Webhook تم إعداده بشكل صحيح
- تأكد من أن الفرع المحدد في Hostinger هو نفسه الذي تدفع إليه التغييرات
- حاول سحب التغييرات يدوياً من لوحة تحكم Hostinger

### المشكلة: أخطاء في الملفات بعد النشر
الحل:
- تأكد من أن جميع الملفات تم رفعها بشكل صحيح
- تحقق من أسماء الملفات (حساسة لحالة الأحرف)
- تأكد من أن جميع المسارات في ملفات HTML و CSS صحيحة

## نصائح إضافية
1. قم بعمل نسخة احتياطية من موقعك قبل إجراء أي تغييرات كبيرة
2. استخدم فروعاً (branches) منفصلة للتطوير والاختبار
3. قم باختبار التغييرات محلياً قبل دفعها إلى GitHub
4. استخدم رسائل واضحة للالتزامات (commits) لتتبع التغييرات بسهولة

## الدعم الفني
إذا واجهت أي مشاكل، يمكنك:
- مراجعة وثائق GitHub الرسمية
- التواصل مع دعم Hostinger
- التحقق من سجلات الأخطاء في لوحة تحكم Hostinger
